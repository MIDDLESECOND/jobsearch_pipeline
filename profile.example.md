# Candidate Profile (gate-relevant facts) — EXAMPLE TEMPLATE

Copy this to `profile.md` and replace the bracketed values with your own. `profile.md`
is gitignored so your personal details stay local. The evaluator reads this verbatim, so
write it as a set of crisp, gate-relevant facts — what would PASS or FAIL each hard gate,
not a resume. The values below are illustrative (a low-code AI / automation candidate).

- **Experience — total:** [N] years ([degree, school, grad date]; [employer, dates]). State
  where a "5 yr min" is a stretch vs. where "6+" is a hard years-floor FAIL.
- **Experience — by function (measure years requirements against the MATCHING tenure, not
  the total):** [one bullet per title/function you've held, with dates and tenure — e.g.
  "[earlier title], [start]–[end] ([N] yrs)" then "[current title], [start]–present ([M] yrs
  in title)". State explicitly which requirement phrasings measure against which tenure; a
  recent title change means "N yrs as [new title]" is a wall the total years hide].
- **Formal people leadership:** [NONE, or your actual tenure with direct reports]. If none,
  say so explicitly — a required "N+ yrs of leadership/management" is then a cold-screen
  wall (the guide's leadership check caps it to RECRUITER_ONLY). Distinguish from
  stakeholder partnership / project leadership, which don't count.
- **Current title:** [your title] at [employer type, not necessarily the name].
- **Production stack (real, defensible):** [languages, platforms, tools you can defend in
  an interview — e.g. Power Automate, AI Builder, Power BI, Python, SQL, REST integration, ETL].
- **Production AI evidence:** [your strongest shipped AI artifact, described honestly —
  e.g. an AI Builder + Power Automate classification system. Note whether it is applied AI /
  prompt engineering / integration vs. from-scratch model development].
- **In-progress / exposure (not production):** [work that counts as exposure, not delivery].
- **Coursework-only (NOT production; ramp-able category):** [tools you know but haven't
  shipped — e.g. LangChain, agent frameworks, RAG, vector DBs. Listed as *preferred* = fine;
  listed as a *core requirement with years attached* = tool-requirement FAIL].
- **Work authorization:** [your status]. State exactly what is a work-auth FAIL (e.g.
  citizenship-only or clearance requirements) vs. PASS (e.g. "authorized without sponsorship").
- **Employment type:** [what you'll accept — e.g. permanent full-time only]. State which
  engagement types are an employment-type FAIL (contract, contract-to-hire, temp, part-time,
  internship, staffing-agency W2-contract).
- **Target roles — primary tier:** [the titles you actually want].
- **Target roles — acceptable tier:** [fallback titles].
- **Explicitly OUT of scope:** [role substance you will not pursue regardless of title —
  e.g. from-scratch model training/tuning, evals/benchmarks, published research].
- **Location:** [e.g. relocation-willing; on-site/hybrid/remote all acceptable]. Note whether
  location is ever a gate fail.
- **Industry:** [open, or list constraints]. Distinguish industry *subject* (usually fine)
  from a stated requirement for prior *sector experience* (a gate fail).
- **Languages:** [if relevant].
